# T-022 Config Schema

Primary config path:

- `~/.config/sec_bootstrapper/config.yaml` (generated by `sec-bootstrapper init`)
- project example: `config/config.yaml`

## Top-Level Keys

- `execution`
  - `phase`: `local_prep|server`
  - `stage`: `stage1|stage2|stage3`
  - `dry_run`: bool
  - `auto_rollback`: bool
  - `log_level`: string
- `target`
  - `host`, `port`, `user`
- `security`
  - `ssh`: port/auth/user list and limits
  - `firewall`: UFW policy and IPv6 toggle
  - `fail2ban`: jail values
  - `auto_updates`: enable/disable
- `system`
  - `timezone`, `entropy_daemon`
- `tailscale`
  - `enabled`
- `docker`
  - `enabled`, `userns_remap`, `gvisor`, pool settings
- `optional`
  - coarse feature toggles (`tailscale`, `docker`)
- `modules`
  - per-module enable flags
- `ai_frameworks`
  - default installer target + framework-specific config
- `stage_gate`
  - `state_file` for persisted gate status
- `tool_cache`
  - `cache_root` (default `/tools`)
  - `fallback_root`
  - `manifest_file`
  - `allow_download`
- `metadata`
  - arbitrary ticket/operator metadata

## Tool Cache Manifest Schema

`tool_cache.manifest_file` points to YAML:

```yaml
tools:
  tool_name:
    version: "1.2.3"
    sha256: "<hex>"
    file: "bin/tool"
    url: "https://..."
    version_cmd: "{path} --version"
```
